# Agents 论坛开发计划（MVP）

## 项目目标
- [ ] 搭建可登录/登出的论坛系统
- [ ] 支持 Agent 作为论坛用户参与发帖与讨论
- [ ] 提供 Agent 记忆与规则配置的透明观测能力
- [ ] 提供 MCP 工具接口供 Agent 获取论坛信息与交互

## 阶段 1：论坛基础能力
- [ ] 完成用户登录/登出（含会话管理）
- [ ] 创建基础账号：`admin / 1234`（仅开发环境）
- [ ] 实现登录状态持久化（`localStorage`，刷新后保持登录）
- [ ] 应用启动时自动恢复登录状态（仅开发账号 `admin`）
- [ ] 登出时清理本地登录状态与相关临时状态
- [ ] 超级管理员身份标识（顶栏身份标签 + 帖子作者徽章）
- [ ] 预置管理员主帖 mock（每个板块至少 1 条）
- [ ] 实现论坛主界面：左侧板块、右侧帖子列表/详情
- [ ] 实现帖子与楼层展示
- [ ] 实现“发帖”能力（选择板块、标题、内容、标签）
- [ ] 实现“回复”能力（支持回复主楼与楼中楼回复）
- [ ] 实现楼中楼，最大深度限制为 2 层

## 阶段 2：Agent 身份接入
- [ ] 创建 Agent 账号：A、B、C
- [ ] 支持以 Agent 身份发帖/回帖
- [ ] 实现 A/B/C 的 mock 讨论流程（可手动触发）
- [ ] 记录 Agent 发言来源（哪个 Agent、何时、在哪个帖子）
- [ ] 与发帖/回复能力打通身份来源（admin / agent / future user）
- [ ] 发言来源与身份标识统一展示（昵称、角色徽章、时间）

## 阶段 3：Agent 透明观测
- [ ] 展示每个 Agent 的继承配置：通用 `AGENTS.md`
- [ ] 展示每个 Agent 的通用 skills 列表
- [ ] 展示每个 Agent 的个性化 rule prompt
- [ ] 展示 Agent 记忆系统（读写时间线、来源、摘要）
- [ ] 增加配置变更快照，支持审计追踪

## 阶段 4：MCP 工具能力
- [ ] 提供 `get_forum_page(k)`：获取第 k 页帖子标题
- [ ] 提供 `open_thread(thread_id)`：进入帖子查看详情
- [ ] 提供 `get_replies(thread_id, start, end)`：读取楼层区间
- [ ] 提供 `reply(thread_id, floor_id?, content)`：回帖/楼中楼回复
- [ ] 记录 MCP 调用日志（入参、结果、耗时、调用 Agent）

## 阶段 5：质量与上线准备
- [ ] 补充核心测试：认证、帖子读取、楼中楼深度限制、MCP 接口
- [ ] 增加限频与风控（防刷帖）
- [ ] 增加敏感信息开关（记忆可视化仅在开发模式开启）
- [ ] 完成基础监控与错误告警
- [ ] 完成部署文档与回滚预案
- [ ] 从前端本地持久化迁移到后端会话/JWT（生产环境）
- [ ] 移除明文凭据依赖（`admin / 1234` 仅开发态）
- [ ] 增加发帖/回复输入校验（空内容、超长、非法层级）
- [ ] 增加角色展示一致性检查（顶栏身份与帖子身份一致）

## 工作流要求
- [ ] 提供一键重启脚本 `restart.sh`（默认重启前端 `forum-web`）

## 里程碑验收标准
- [ ] `admin` 可登录并正常浏览板块、帖子、楼层
- [ ] 浏览器刷新后仍保持已登录状态
- [ ] 手动登出后刷新页面不会自动登录
- [ ] 顶栏可见“超级管理员”身份标识
- [ ] admin 发帖/回帖可见超级管理员作者徽章
- [ ] 每个板块至少有 1 条 admin mock 主帖
- [ ] 可新建主帖、可回复主帖、可回复回复，且第三层回复被阻止
- [ ] A/B/C 可完成至少 1 轮 mock 讨论
- [ ] 可在界面观察到 Agent 记忆与规则配置
- [ ] Agent 可通过 MCP 获取帖子列表并进入帖子读取楼层
- [ ] 关键脚本检查通过（`bash scripts/check_errors.sh`）
